# RecruitIQ — Implementation Plan

## Current State Assessment

**Backend** (Express/TypeScript) already has:
- Qdrant vector DB client, collection, insert/search/retrieve
- Qwen LLM client via OpenRouter
- JD parsing, candidate ranking (skill/experience/education matchers)
- Candidate comparison, search with filters
- Logging, caching, error handling middleware
- Supabase credentials in `.env`, seed script exists but **no Supabase integration code**

**Frontend** (Next.js 16 App Router) already has:
- Home page, candidate search, candidate detail, compare, how-it-works pages
- UI components (cards, badges, sheets, buttons, skeletons)
- Zustand store with persistence
- API client module, Next.js proxy rewrites to backend

**Missing entirely** (the 4 Pillars):
- Supabase Storage/DB client code
- `upload_sessions` management
- Public resume upload page
- Resume PDF/DOCX parsing
- Flight risk / growth trajectory analysis
- AI screening questions generation
- AI closing strategy generation
- Recruiter dashboard with link generation

**SQL to run in Supabase editor:**
```sql
CREATE TABLE upload_sessions (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  job_description_text TEXT NOT NULL,
  created_at TIMESTAMP DEFAULT NOW()
);

ALTER TABLE candidates ADD COLUMN IF NOT EXISTS upload_session_id UUID REFERENCES upload_sessions(id);
ALTER TABLE candidates ADD COLUMN IF NOT EXISTS flight_risk TEXT CHECK (flight_risk IN ('Low','Medium','High'));
ALTER TABLE candidates ADD COLUMN IF NOT EXISTS growth_trajectory TEXT CHECK (growth_trajectory IN ('Fast-track','Steady','Stagnant'));
ALTER TABLE candidates ADD COLUMN IF NOT EXISTS parsed_json JSONB;
```

**Supabase Storage:** Create `resumes` bucket with RLS: allow public inserts (for candidate uploads), allow service_role full access.

## Phase 2: Generate Link API + Upload Pipeline

| # | File | Action | Description |
|---|------|--------|-------------|
| 5 | `backend/package.json` | Edit | Add `multer`, `pdf-parse`, `mammoth`, `@types/multer` |
| 6 | `backend/src/controllers/sessionController.ts` | Create | `generateLinkHandler`: POST -> creates `upload_sessions` row, returns `{link: '/upload/' + uuid}` |
| 7 | `backend/src/controllers/uploadController.ts` | Create | `uploadResumeHandler`: receives file via multer -> saves to Storage -> extracts text -> sends to Qwen -> inserts to DB + Qdrant |
| 8 | `backend/src/services/resume-parser.ts` | Create | `extractTextFromPdf()` using `pdf-parse`, `extractTextFromDocx()` using `mammoth` |
| 9 | `backend/src/services/llm/parseResume.ts` | Create | Qwen prompt to extract: name, email, phone, skills, experience, education, companies -> returns structured JSON |
| 10 | `backend/src/services/llm/flightRisk.ts` | Create | Qwen analyzes tenure per job -> `flight_risk` (Low/Med/High) + `growth_trajectory` (Fast-track/Steady/Stagnant) |
| 11 | `backend/src/routes/sessionRoutes.ts` | Create | `POST /generate-link`, `GET /sessions/:id` |
| 12 | `backend/src/routes/uploadRoutes.ts` | Create | `POST /upload/:uuid` (public, no auth, multipart) |
| 13 | `backend/src/routes/index.ts` | Edit | Mount new route groups |
| 14 | `backend/src/server.ts` | Edit | Add multer error handling |

**Upload Pipeline Flow:**
1. Multer receives file -> validates PDF/DOCX
2. Save to Supabase Storage `resumes/{uuid}/{timestamp}_{filename}`
3. Extract text: `pdf-parse` for PDF, `mammoth` for DOCX
4. Call Qwen `parseResume(rawText)` -> structured JSON
5. Call Qwen `flightRisk(rawText, parsedJson)` -> flight_risk + growth_trajectory
6. Insert candidate into Supabase `candidates` table
7. Insert skills into `candidate_skills`
8. Generate embedding via existing `generateEmbedding()` -> upsert to Qdrant
9. Return success
